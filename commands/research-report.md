# /research-report — 자동 리서치 보고서 생성

너는 이 워크플로의 **오케스트레이터**다. 사용자가 준 주제: **$ARGUMENTS**

주제가 비어 있으면 먼저 한 줄로 주제를 물어보고 멈춘다.

아래 단계를 순서대로 수행한다. 서브에이전트는 Task 도구로 호출한다.

## 1단계 — 주제 분해
주제를 3~6개의 하위 질문으로 분해한다. 각 질문이 내부 자료로 답할 것인지, 웹으로
답할 것인지(또는 둘 다) 표시한다. 분해 결과를 사용자에게 짧게 보여준다.

## 2단계 — 병렬 조사
조사는 서로 독립적이므로 **한 번에 병렬로** 호출한다(같은 메시지에서 Task 다중 호출):
- 내부 자료가 필요한 질문 → `internal-researcher` 에게 위임
- 외부/최신 정보가 필요한 질문 → `web-researcher` 에게 위임

각 에이전트에는 담당 하위 질문을 명확히 전달한다.

## 3단계 — 종합
모든 조사 결과를 모아 `synthesizer` 에게 넘겨 보고서 구조(들어가며(서문) +
목차 + 섹션별 head/본문/출처)로 통합한다.

## 4단계 — deck 생성
종합 결과를 `ppt-writer` 에게 넘긴다. ppt-writer는 ppt-template 스킬 규칙대로
JSON spec을 만들고 `build_deck.py` 를 실행해 `report.pptx` 를 생성한다.

## 5단계 — 보고
최종 deck의 절대 경로, 슬라이드 수, 간단한 검수 결과(출처 누락/보완 필요 항목)를
사용자에게 보고한다.

## 주의
- 조사 단계(2)는 반드시 병렬로. 종합·생성(3~4)은 순차로.
- 출처 없는 주장은 deck에 넣지 않는다.
- 리서치 자료 폴더는 `RESEARCH_DIR` 환경변수로 설정돼 있어야 한다(없으면 안내).
